Зафиксировать продуктовые требования к данным для базовых сценариев клиента и администратора (vision §4) в отдельном документе с матрицей «сценарий → сущности и данные».

## Шаги

1. Сверить перечень сценариев с [docs/vision.md](../../../../vision.md) §4 и [docs/tech/data-model.md](../../../../tech/data-model.md).
2. Подготовить [docs/tech/user_scenarios.md](../../../../tech/user_scenarios.md): каналы бот/веб, матрицы для клиента и админа, сквозные сценарии, открытые вопросы.
3. Добавить в [docs/tech/data-model.md](../../../../tech/data-model.md) отсылку к документу требований как к источнику сценариев.
4. Обновить [docs/plan.md](../../../../plan.md) (этап 1): выходной артефакт и при необходимости факт реализации.
5. Отметить итерацию выполненной в [tasklist-database.md](../../../tasklist-database.md).
