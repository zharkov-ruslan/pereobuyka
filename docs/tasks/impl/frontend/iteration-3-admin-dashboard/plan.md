# План iter-fe-03 — экран «Панель администратора»

## Цель

Реализовать рабочий маршрут `/admin`: KPI на сегодня, сетку слотов текущей недели, простую недельную статистику и быстрые действия по записи/визиту через backend API iter-fe-01.

## Решения

- **Источник данных:** только backend API: `GET /api/v1/admin/dashboard/today`, `GET /api/v1/admin/dashboard/week-grid`, `GET /api/v1/admin/analytics/week`.
- **Auth:** использовать текущую MVP-сессию из `localStorage` и Bearer token из iter-fe-02; серверная защита данных остаётся на backend.
- **UI:** shadcn/ui (`Card`, `Badge`, `Button`, `Sheet`, `Field`, `Input`) и семантические Tailwind-токены; без новой библиотеки графиков, статистика отображается компактными таблицами/полосами.
- **Действия:** отмена записи через `PATCH /api/v1/admin/appointments/{id}`, подтверждение визита через `POST /api/v1/admin/visits`, оценка клиента через `POST /api/v1/admin/visits/{id}/client-rating`.
- **Границы:** редактирование состава услуг/скидки и полноценная клиентская карточка остаются для iter-fe-04; в этой итерации экран закрывает обзор и быстрые действия по слотам.

## Состав работ

1. Добавить типизированный клиент `web/lib/admin-api.ts` для данных панели и мутаций.
2. Заменить заглушку `/admin` на интерактивный client component:
   - loading/empty/error;
   - KPI карточки и источники записей;
   - сетка недели пн-вс со слотами и событиями;
   - sheet с деталями выбранного события;
   - кнопки отмены/подтверждения/оценки клиента;
   - недельная статистика и топ услуг.
3. Обновить `docs/tasks/tasklist-frontend.md`, создать `summary.md`.
