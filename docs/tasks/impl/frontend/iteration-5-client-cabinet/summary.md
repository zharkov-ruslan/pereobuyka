# Итог iter-fe-05 — личный кабинет клиента + визард записи

## Реализовано

- Добавлен полноценный маршрут `/client`:
  - приветствие клиента, бонусный баланс и CTA «Запись в сервис»;
  - переход в Telegram-бот через `NEXT_PUBLIC_TELEGRAM_BOT_URL`;
  - список записей с отменой scheduled-записи и скрытием отменённых по умолчанию;
  - список визитов с оценкой сервиса и комментарием.
- Добавлен визард записи на той же странице:
  - мультивыбор услуг из `GET /api/v1/services`;
  - расчёт длительности, стоимости, бонусов к списанию и начислению;
  - поиск свободных слотов через `GET /api/v1/slots`;
  - создание записи через `POST /api/v1/appointments`.
- Добавлен `localStorage`-черновик `pereobuyka:web:appointment-draft:v1`; после успешного создания записи черновик очищается.
- Добавлен клиентский API-слой `web/lib/client-api.ts`.
- В `web/.env.example` добавлена переменная `NEXT_PUBLIC_TELEGRAM_BOT_URL`.

## Решения и ограничения

- Итоговый расчёт записи остаётся на backend; frontend показывает предварительный расчёт по каталогу и правилам лояльности.
- Количество каждой выбранной услуги в визарде фиксировано как `1`, что соответствует текущему MVP-каталогу.
- Контроль роли клиента остаётся на backend; UI дополнительно проверяет локальную MVP-сессию.
