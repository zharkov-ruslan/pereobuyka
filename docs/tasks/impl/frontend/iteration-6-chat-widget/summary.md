# Итог iter-fe-06 — чат с ассистентом

## Сделано

- Добавлен `web/lib/consultation-api.ts`: выборка всей истории с постраничным `GET`, `POST` сообщения с опциональной передачей `history` (последние до 20 реплик из локальной ленты).
- Переписан `web/components/chat-widget.tsx`: управляемый Sheet, тёмная панель (zinc/emerald), пользователь справа, ответы ассистента слева с иконкой `Bot`, скролл к низу при изменении ленты, индикатор на исходящем сообщении во время запроса.
- Ошибки: для `503` и прочих 5xx при отправке показывается нейтральная формулировка «Консультант временно недоступен…»; для `422` подставляется безопасное сообщение из контракта `error.message`.

## Отклонения от плана

- Отдельный focus trap не добавлялся — используется поведение диалога из `@base-ui/react` через обёртку Sheet.
- Загрузка истории вызывается из `onOpenChange` панели (а не из `useEffect`), чтобы не нарушать правило `react-hooks/set-state-in-effect` при включённом ESLint.

## Ручная проверка

1. Войти как клиент (или админ с валидным `mvp-*` токеном).
2. Открыть чат → убедиться, что история подтягивается (или пустое состояние).
3. Отправить сообщение → ответ появляется после сохранения на сервере; после перезагрузки страницы история сохраняется.
