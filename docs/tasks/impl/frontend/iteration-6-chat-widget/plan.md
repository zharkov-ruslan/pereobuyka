# План iter-fe-06 — чат с ассистентом (плавающий виджет)

## Цель

Заменить заглушку в `ChatWidget`: загрузка истории `GET /api/v1/consultation/messages`, отправка `POST /api/v1/consultation/messages`, тёмный «терминальный» UI по [ui-requirements.md](../../ui/ui-requirements.md).

## Решения

- **Auth:** Bearer из `loadAuthSession()` (тот же `mvp-*` токен, что и для остальных API).
- **История:** пагинация на клиенте циклом `limit=100` до `total`; при открытии панели — полная перезагрузка списка.
- **После отправки:** тихий refetch истории без спиннера на всю ленту; исходящее сообщение с индикатором ожидания до ответа сервера.
- **Ошибки LLM / 503:** пользователю показывается обобщённое сообщение, без текста исключений провайдера из тела ответа.
- **Доступность:** у шторки стандартное закрытие Base UI Dialog (в т.ч. фокус и Esc через компонент Sheet); поле ввода получает фокус после загрузки истории; регион сообщений с `aria-live="polite"`.

## Файлы

- `web/lib/consultation-api.ts` — запросы к consultation API.
- `web/components/chat-widget.tsx` — UI и состояние виджета.

## Проверки

- `pnpm run lint`, `pnpm run build` в `web/`.
