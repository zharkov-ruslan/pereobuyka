# ADR-006: Вопросы администратора к операционным данным (ограниченный NL→SQL)

**Статус:** Accepted  
**Дата:** 2026-05-02  

## Контекст

Администратору нужны ответы на произвольные аналитические вопросы по данным системы (записи, визиты, клиенты), без ручного SQL и без доступа клиента к этому каналу.

Риски:

- Произвольный SQL из LLM → injection, DDL/DML, утечки через тяжёлые запросы или побочные эффекты.
- Персональные данные в выборках и в логах.

## Рассмотренные варианты

1. **Полный Text-to-SQL без ограничений** — отклонён: неприемлемый риск при ошибке модели или промпта.
2. **Только заготовленные шаблоны запросов + слоты параметров** — очень безопасно, но хрупко при расширении вопросов; высокая стоимость сопровождения.
3. **NL→SQL с жёсткой валидацией AST + белый список таблиц + обёртка LIMIT + таймаут** — баланс гибкости и контроля (выбрано).
4. **Отдельный read-only DSN / роль PostgreSQL** — усиливает defense-in-depth; для текущего масштаба опционально (можно добавить позже без смены контракта API).

## Решение

- Эндпоинт **только для администратора:** `POST /api/v1/admin/analytics/data-insight`, Bearer `ADMIN_API_TOKEN`.
- Модель генерирует один **SELECT** по текстовому описанию схемы `public` (операционные таблицы проекта).
- Перед выполнением запрос проходит проверку **sqlglot**: одна инструкция, корень — SELECT (включая UNION / WITH … SELECT), запрет INSERT/UPDATE/DELETE/DDL и опасных функций, **белый список имён таблиц**, только схема `public`.
- Выполнение: обёртка `SELECT * FROM (<inner>) AS _nlq_sub LIMIT N+1`, параметры **`statement_timeout`** и **`admin_nl_sql_max_rows`** из конфигурации.
- **Аудит:** в серверный лог пишутся `admin_user_id`, текст вопроса (обрезанный) и нормализованный inner SQL (без секретов).
- Второй вызов LLM формирует **краткое резюме** по фактическим строкам выборки (без выдумывания чисел).

Клиентский чат и клиентские токены к этому эндпоинту **не** допускаются.

## Последствия

- Зависимость **sqlglot** для парсинга и нормализации SQL.
- При смене схемы БД нужно обновлять белый список таблиц и текст схемы в `safe_nl_sql.py` / промпте.
- При необходимости усиления — добавить роль `readonly` в PostgreSQL и второй DSN только для этого пути.
