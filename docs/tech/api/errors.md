# Ошибки API «Переобуйка»

Контракт тела ошибки (OpenAPI: `ErrorResponse` / `error`):

```json
{
  "error": {
    "code": "string",
    "message": "string",
    "details": {}
  }
}
```

- **code** — стабильный машинный код (для клиентов и логики).
- **message** — краткое описание для человека; не дублировать секреты и внутренние детали интеграций.
- **details** — опционально: поля валидации, конфликтующий ресурс и т.п.

## HTTP-статусы

| Статус | Когда |
|--------|--------|
| 400 | Некорректное тело или параметры (не 422) |
| 401 | Нет или невалидный токен |
| 403 | Токен есть, но роли/прав недостаточно |
| 404 | Ресурс не найден |
| 409 | Конфликт домена (слот занят, неверный переход статуса) |
| 422 | Ошибки валидации схемы (Pydantic / OpenAPI) |

## Доменные коды `error.code` (примеры)

| code | Типичный HTTP | Смысл |
|------|----------------|--------|
| `slot_unavailable` | 409 | Выбранное окно уже занято или не входит в расписание |
| `service_inactive` | 400 / 404 | Услуга отключена или недоступна для записи |
| `appointment_not_cancellable` | 409 | Отмена запрещена правилами (например, слишком поздно) |
| `insufficient_bonus_balance` | 409 | Недостаточно бонусов для списания |
| `bonus_spend_limit_exceeded` | 409 | Списание бонусов превышает лимит % от суммы |
| `visit_already_confirmed` | 409 | Визит по записи уже закрыт |
| `conflict` | 409 | Общий конфликт версии/состояния |

Точный реестр кодов поддерживается в [`openapi.yaml`](openapi.yaml) (описания ответов 4xx у операций) и по мере реализации — в коде backend.
