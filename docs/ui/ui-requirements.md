# Требования к интерфейсу

Стиль похож на https://www.tbench.ai/

## Экран 1: Панель администратора сервиса

Роль пользователя: администратор сервиса.

Экран при открытии должен сразу давать возможность увидеть загруженность сервиса на текущий день - без кликов, без поиска.

**Вверху - суммарная информация:**
Блоки с ключевыми показателями загруженности сервиса:
- сколько записей на сегодня всего;
- сколько визитов за сегодня уже было;
- сколько отменено визитов на сегодня;

- сколько записано на визит сегодня через ИИ-консультанта в Telegram/Web, меню чат-бота в Telegram, клиентский web-интерфейс и через администратора;
- сколько вопросов задано боту за последнюю неделю;
Цифры можно сгруппировать для оптимизации пространства.

**Расписание активности по слотам на текущую неделю:**
Под цифрами — сетка расписания загруженности сервиса на каждый день (столбцы) текущей недели (пн-вс) по слотам (строки) с отметками:
- есть запись - оранжевый цвет;
- визит состоялся (подтвержден) - зеленый цвет;
- запись отменена - сервый цвет, если на это время нет другой записи или подтвержденного визита.
При наведении на каждую клетку с записью/визитом/отменой отображать в виде тултипа список услуг, общую стоимость.

Если кликнуть по слоту с записью, то должно открыться модальное окно с:
- деталями записи: имя клиента, телефон, список услуг с временем по каждой услуге и стоимости каждой услуги, общим временем и стоимостью без скидки и со скидкой и с количеством начисленных бонусов, если визит будет подтвержден;
- кнопками отмены записи и подтверждения визита.

При клике на кнопку подтверждения модальное окно с деталями записи должно закрыться и открыться небольшое окно с оценкой клиента администратором в виде 5 звезд и полем для комментария, чтобы администратор мог оценить клиента и при необходимости оставить комментарий.

На экране в отдельной вкладке должна быть возможность посмотреть статистику за текщую неделю:
- топ-3 самых востребованных услуг;
- записей, визитов и отмен записей на одном графике;
- график с доходом за каждый день недели;
- количества записей на услуги через LLM, через меню бота и администратора за каждый день недели.


## Экран 2: Клиенты сервиса

Роль пользователя: администратор сервиса.

**Таблица клиентов:**
Список клиентов в алфавитном порядке. У каждого — количество визитов, общая потраченная сумма, текущее кол-во бонусов, рейтинг клиента от 1 до 5 с точностью до сотых.

**Карточка клиента:**
При клике по строке таблицы с клиентом должна открыться форма карточки клиента на той же странице. Сверху должна быть ссылка "К списку клиентов" для возврата к таблице всех клиентов. На карточке клиента должна отображаться информация:
- Имя;
- телефон;
- признак связи через Телеграм (иконка Телеграма), если есть TelegramID клиента. По клике на иконку телеграма должен открываться Телеграм и чат с этим клиентом;
- рейтинг клиента;
- вкладка с предстоящими записями клиента, в т.ч. отмененными (по умолчанию должны быть скрыты), со списком требуемых услуг, их стоимостями, длительностями, общей стоимостью и общей длительностью, а также кнопки подтверждения и отмены записи; При клике на кнопку подтверждения должно открываться модальное окно с оценкой клиента сервисом, как описано выше. Должна быть возможность отредактировать запись в сервис: добавить/удалить услугу, изменить размер скидки и расчитать начисляемое количество бонусов, общую сумму за предстоящий визит;
- вкладка с подтвержденными визитами клиента со списком оказанных услуг, их стоимостью, общей стомостью за визит, процентом скидки, оплаченной клиентом суммой и количеством начисленных бонусов, оценкой и комментарием (при наличии)от сервиса по факту оказания услуги. Должна быть возможность отредактировать данные о визите: добавить/удалить услугу, пересчитать общую стоимость услуг, количество начисленных и списанных баллов.


## Экран 3: Личный кабинет клиента

Роль пользователя: клиент сервиса.

Клиент заходит и видит:
- приветствие по имени, ниже кнопка "Запись в сервис" для самостоятельного выбора требуемых услуг и записи в сервис и кнопка для перехода в телеграм бот;
- кнопку "Запись в сервис", при нажатии на которую должна открыться форма-визард на той же странице с выбором услуги или нескольких услуг из списка, расчетом общей длительности на визит, общей стоимости, количества начисляемых баллов за визит. Сверху визарда должны быть ссылка "На главную" для выхода из визарда. Все, что успел набрать клиент в визарде, должно сохранться в LocalStorage браузера и предзаполняться при слудющем открытии визарда (сделать черновик записи и удалять его при подтверждении записи клиентом);
- список записей в сервис, в т.ч. отменных (скрыты по умолчанию), с подробностями по записи и возможностью отменить запись;
- список визитов в сервис с подробностями по визиту. У клиента должна быть возможность поставить оценку сервису 1-5 звезд по каждому визиту с указанием комментария (опционально).


## Чат с AI-ассистентом (только роль: клиент)

Чат показывается **только** в клиентском веб-контуре (личный кабинет и связанные экраны клиента): не отдельная страница, а плавающая кнопка в углу. Нажал — открылась панель поверх текущего контента, закрыл — вернулся туда, где был.

**Административный контур** этот чат **не** использует: у админа отдельные инструменты (например, запросы к данным / отчётность на панели), без плавающего диалога LLM-консультанта как у клиента.

Внутри панели: история переписки сверху, поле ввода снизу. Старые вопросы и ответы видны, можно прокрутить вверх. Новый вопрос — пишешь, отправляешь, ответ появляется с небольшой задержкой.

Визуально напоминает терминальный чат или IDE-ассистент: тёмный фон, сообщения пользователя справа, ответы ассистента слева с иконкой.
